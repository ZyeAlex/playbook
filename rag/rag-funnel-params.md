# RAG 检索漏斗与证据裁剪

## 适用场景

规则驱动文档审核：检索 → 重排 → LLM 判定 → 可复核证据落库。

## 推荐漏斗

| 阶段 | 参数 | 要点 |
|------|------|------|
| scope 预过滤 | 章号 path + 表号 | 不用正文关键词 |
| hybrid | top-k 30–40 | child 级 |
| parent 去重 | — | 同 parent 只留代表 child |
| rerank | top-k 8–12 | 再映 parent |
| hybrid 救援 | 1–2 槽 | parent 级，未进 rerank |
| 合并 | 救援在前 | 防 top_k 截断 |
| LLM 池 | 6–10 parent | |
| 表附带 | ≤3 张 | 段内「表 N-M」→ 挂表块 |
| 落库裁剪 | ≤3 块 | LLM 引用后 prune |

## 门控

| 项 | 典型值 |
|----|--------|
| 检索分下限 | 0.35 |
| 综合分 | 0.55·LLM + 0.30·检索 + 0.15·无占位 |
| Review 阈值 | 0.75 |

语义门控（向量算规则-证据相似度）：仅低 LLM 置信时降级，**不必展示给用户**。

## 检索 query 构建

只用规则结构化字段（名/述/场景/标签），**不注入**文档 mock 关键词或表头扩展词。

## prune 逻辑（LLM 引用 &gt;3）

1. path 层级去重（同小节留高分 1 条）
2. 保留 quote 锚点块
3. 最高分段落 + 整表各 1
4. 按 rerank 分填满至上限 3

## 反模式

- 检索 top-k 回填 evidences
- 正文关键词表过滤
- 无上限加大 top-k「试试能不能召回」

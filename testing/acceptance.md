# 验收与回归分层

## 适用场景

文档 RAG 类产品上线前个人验收与持续回归。

## 分层

| 层 | 内容 | 频率 |
|----|------|------|
| 单元 | 解析、切片、prune、scope | 每次改逻辑 |
| 管道 | 表附带、救援合并、golden 锚点 | 改 RAG 参数 |
| 续跑 | 跳过已完成、bootstrap 重调度 | 改 worker |
| 业务 | 固定样例 docx + 规则集全量跑 | 大版本 |
| 个人 | 分层抽检 + 源文档定位 | 发版前 |

## 个人验收步骤

1. 确认模型本地就绪，服务正常启动
2. 上传**固定样例文档** + 选定规则集，SSE 至 100%
3. 记录 KPI 分布（Pass/Partial/Fail/Review/N/A）
4. 每类抽检 ≥1 条：读 reason → 打开源文档预览核对高亮
5. 记录局限（规则集是否过宽、占位表题等）

## 批量测试注意

规则集混入**与文档场景无关**的条目 → Fail 偏高是预期，不代表检索全失效。应用「与文档匹配」的子集评 RAG。

## 自动化原则

- golden case 锚定「应召回 blockId」，不绑死 LLM 文案
- 业务对比脚本：DB 存量 vs 可选重跑 RAG，用于参数实验

## 反模式

- 只看 KPI 不点开源文档
- 用过大规则集否定整体准确率

## 项目来源

文档审核发版验收，2026 年验证。

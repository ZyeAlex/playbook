# 模式：规则审文档（RAG + LLM 判定）

> 适用于「检查单/规则库 + 被审 Word/PDF + 自动判定 + 人工复核」类产品。  
> 各工具细节见同目录其他原子文件。

## 推荐架构

```
规则库 ──快照──► 审核任务
被审文档 ──解析──► 块索引 ──检索──► 证据池 ──LLM──► 判定 + 引用块
                                      └──► SSE 进度 ──► 前端预览高亮
```

## 通用硬约束（验证过）

| 约束 | 理由 |
|------|------|
| 执行期只读**规则快照** | 避免审到一半规则被改 |
| 落库证据**仅**来自 LLM 声明的块 ID | 检索候选≠证据，否则滥引 |
| 落库证据 **≤3 块** | 控制 token、可复核、防 LLM 凑数 |
| 不用正文关键词表删证据 | 易误杀；scope 用章号/表号等**结构**信息 |
| 解析/索引**版本号**进缓存 key | 版本升级必须重跑，否则 silent wrong |
| 重模型任务 **单 worker** 或显式队列 | 防双实例同时加载向量+重排模型 OOM |

## 索引两层（Parent-Child）

| 层 | 粒度 | 用途 |
|----|------|------|
| Child | 单段 / 整表 | 向量检索、rerank |
| Parent | 同章节多段合并 | 送 LLM、引用 ID |

表行级块仅用于溯源高亮，**不进**向量索引（与表块重复）。

## 检索漏斗（经验参数区间）

| 阶段 | 典型值 | 备注 |
|------|--------|------|
| hybrid top-k | 30–40 | 再小易漏；再大挤占后续池 |
| rerank top-k | 8–12 | |
| hybrid 救援槽 | 1–2 | 未进 rerank 的高分 parent 兜底 |
| LLM 证据池 | 6–10 parent | + 段内「表 N-M」附带表 ≤3 |
| 检索门控 | ~0.35 | 归一化后 |
| 综合门控 | 0.55·LLM + 0.30·检索 + 0.15·质量 | 阈值常用 0.75 |

盲目加大 top-k 常导致**表附带配额被占满**，整体反而变差。

## 置信度分层

| 展示 | 来源 | 用途 |
|------|------|------|
| 规则 LLM 分 | Judge 整体 confidence | UI 规则行 |
| 块 LLM 分 | evidenceScores 每块 | UI 证据行 |
| 综合分 | 门控公式 | 自动降级 Review、导出 |

## 前端预览与后端块的关系

- 后端块有稳定 ID（`P-*`、`TBL-*`、`PU-*`）
- 浏览器 docx-preview **无** blockId DOM → 高亮靠**文本匹配** + `sourceBlockIds` 扩展（见 `frontend/evidence-highlight-dom.md`）

## 反模式

- 用检索 top-8 直接当 evidences 落库
- 按规则名子串做 scope 过滤
- 热重载后假设内存里的后台任务仍存在
- 列表接口快照当详情用（结果列表需全量接口）

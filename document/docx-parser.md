# python-docx 结构化解析

## 适用场景

QTP/大纲类 Word：章节标题、目录、表题、段落、表格 → 统一 `DocumentBlock` 供 RAG。

## 推荐块类型

| type | 索引 | 溯源 |
|------|------|------|
| paragraph | ✓ child | ✓ |
| table（整表） | ✓ child | ✓ |
| table_row | ✗ | ✓ 仅高亮 |

## path 构建

1. Heading 1–4 维护栈 → `path = "章 / 节 / …"`
2. **跳过目录**：「目录」标题 + 行尾 `\t页码` 模式
3. **表题正则**：`表\s*(\d+(?:-\d+)*)`（与「段内表引用」逻辑共用）
4. Caption 样式或正文表题 → 绑定后续表格 path

## 版本字段

解析逻辑变更时 bump `PARSER_VERSION` 字符串，写入索引缓存 key。

## 踩坑

| 问题 | 解法 |
|------|------|
| 目录进向量索引 | TOC 区域 skip |
| 表无章节上下文 | 表题/Caption 挂 path |
| 模板「表X」占位 | 无法映射真实表号 → 段引附带失败（需占位映射方案） |
| 按字数硬切段落（旧） | 改为 Heading 树，避免断章 |

## 反模式

- 忽略 Word 样式只按纯文本切段
- table_row 与 table 同时进索引（重复、挤占 top-k）

## 项目来源

电磁类试验大纲 docx 审核，2026 年验证。

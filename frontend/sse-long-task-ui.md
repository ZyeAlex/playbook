# SSE 长任务前端

## 适用场景

分钟级后台任务（文档审核、批量提取），列表/详情实时进度。

## 端点模式

| 类型 | 用途 |
|------|------|
| 全局 `/events` | 工作台多任务进度条 |
| 单任务 `/{id}/events` | 详情页进度 + 子步骤 |

`EventSource` 或 fetch 读 `text/event-stream`，解析 `data: {json}\n\n`。

## 事件类型建议

| type | 字段示例 |
|------|----------|
| `task` | progress, statusMessage, 计数 KPI |
| `step` | 当前条目 index、步骤名 |
| `llm_stream` | 流式字符预览（可选） |

## 详情加载

进入详情 → **全量 GET** 拉结果列表；SSE 只更新进度与增量，不替代全量结构。

## 人工复核

PATCH 单条结果 → 前端乐观更新 + 失败回滚；服务端写变更留痕表（操作人待权限体系）。

## 展示字段经验

- 规则行：Judge 整体置信度
- 证据行：块级置信度
- 一般不向用户展示 rerank 分 / 向量语义分（仅门控用）

## 反模式

- 仅靠 SSE 拼装详情（断线缺数据）
- 列表页不订阅全局 SSE 导致进度 stale

## 项目来源

文档审核 SSE 联调，2026 年验证。
